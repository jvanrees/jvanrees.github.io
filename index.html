Bio

Hello, my name is Jeff and in Winter 2017 I was amongst the first group of graduates from the University of Wisconsin's GIS and Web Map Programming Master’s degree program. This unique program focusses on Cartography, Data Visualization, Development, and GIS and how they work together shaping the future of GIS. I make maps on the internet and I want to help shape the future of mapping with you.

Over the course working on my graduate degree, I’ve gravitated toward open-source technologies. In the traditional GIS sphere GDAL/OGR and QGIS. On the web, mapping and visualization libraries like Leaflet, Mapbox-GL, and D3 help provide interactive experiences.

My personal favorite technologies in the spatial sphere are PostGIS and Mapbox. PostGIS allows for complex, powerful, and fast geospatial processing. Mapbox is worlds ahead of the competition for mapping applications by providing a slick web interface with Mapbox Studio and its use of WebGL to create immersive experiences.

Projects

 

Rocky Mountain Arsenal Visitors Map

The Rocky Mountain Arsenal National Wildlife refuge is one of the nations largest urban wildlife refuges. Here I created a park web application to help guide the user around the refuge.

It is a full stack web application built with Node.js and the Express.js web application framework. It is powered by a PostGIS database backend and Mapbox-GL JS for the front end. Users can search for features and amenities in the park, and submit their own features.

For this project, OpenLayers and GeoServer were recommended. However, I decided to go above and beyond and tackle technologies that were new to me by building the application from the ground up.

Rocky Mountain Arsenal Android App

Building upon the above application. This mobile application will help visitors to the Rocky Mountain Arsenal see the history that has been scraped from the landscape. Users will experience the Arsenal’s history through a timeline and an interactive map showing the old building footprints.

I digitized the buildings in QGIS using old aerial imagery and old hand drawn maps. Gathering the data, digitizing it, and organizing turned out to be a huge task.

I’m always looking towards the latest trends, and excited to lean new technologies. Thus, for this application I chose React Native for Android as the backbone of the application with Mapbox GL React Native powering the maps. These were all new technologies for me, but it demonstrates that I’m unafraid of doing things differently and tackling new challenges.

D3 Coordinated Visualization

Interactivity is one of the strongest attributes of mapping and data visualization on the internet. Here, D3 is used to link a chart with an interactive map. A mouseover event on the map will highlight the corresponding bar in the chart. The user can select different years to see a different set of data, and the chart reorganizes itself from high to low and colors are automatically assigned using D3’s quantile method.

CartoCSS: Qing Dynasty Slippy Map

Old maps are beautiful. For my first ever webmap I was tasked to design a slippy map using CartoCSS in Mapbox Studio Classic. Desiring to create something unique, I decided to recreate the aesthetics of a Qing Dynasty water color map into a webmap. What drew me to this map was the beauty of how the cartographer displayed the water and the mountain ranges.
What I learned most from this project was the importance of zoom level and visualization. Different zoom levels require different levels of detail, and different image sizes. As it was my very first map created to be an artistic piece and an homage to an inspirational piece of cartographic art, I've decided to rework this one. If you love old maps, check out the original Qing dynasty era map here.
